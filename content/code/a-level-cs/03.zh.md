## 出错的时候

- 运行时错误通常会让程序**崩溃**。
- **异常**(exception)是 Python 发出的"出问题了"的信号。
- 异常处理让你捕获它,并继续运行。

---

## try / except

- 把有风险的代码放进 `try:` 代码块。
- 如果它抛出异常,就会运行 `except:` 代码块,而不是崩溃。
- 如果没有出错,`except` 代码块会被跳过。

---

```python
try:
    number = int("not a number")
except ValueError:
    number = 0
print(number)
```

---

## 捕获正确的错误

- 写明你预期的错误**类型**:`except ValueError:`。
- `ValueError` 表示用于转数字的文字有误;`ZeroDivisionError` 表示除以零。
- 捕获具体类型,才不会把别的 bug 也一起隐藏掉。

---

```python
try:
    print(10 / 0)
except ZeroDivisionError:
    print("Cannot divide by zero")
```

---

## finally 与 raise

- `finally:` 代码块**总会**运行,无论是否出错 —— 适合做清理工作。
- `raise` 让**你**主动发出一个错误信号。
- `raise ValueError("message")` 会停止代码,除非有什么东西捕获它。

---

```python
try:
    raise ValueError("bad value")
except ValueError as e:
    print("Caught:", e)
finally:
    print("Always runs")
```

---

## 为什么重要

- 一个健壮的程序不应该因为坏的输入就崩溃。
- 处理异常让它能提醒用户,然后继续运行。
- 它还能确保文件和其他资源被干净地关闭。
