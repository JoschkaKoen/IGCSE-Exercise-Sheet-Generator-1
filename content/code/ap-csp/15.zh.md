## 创作型表现任务

- **创作型表现任务**(Create Performance Task)是一个你自己设计和编写的小程序。
- 在真正的 AP 考试里,你要通过 **AP 数字作品集**(AP Digital Portfolio)提交它(你的代码、一段视频和书面回答)。
- **重要提示:** 真正的任务由 AP 评分,**不是**由本网站评分。这节课只帮助你**练习**并搭好结构。

---

## 你的程序必须包含什么

- 一个**列表**(list,或其他集合),把许多值存放在一起。
- 一个**过程**(procedure,即函数),它带有**至少一个参数**,并且你在代码里**调用**了它。
- 一个**算法**,在**迭代**(循环)**内部**使用**选择**(`if`)。
- 一些**输入**和一些**输出**,让程序做点有用的事。

---

## 挑一个小点子

- 保持简单:你只需要展示上面这些技能,而不必做一个庞大的应用。
- 好点子:给一组分数评级、把购物清单求和、把学生分到不同小组、统计单词数。
- 选你看得懂的数据,这样你之后才能解释它。

---

## 一个起始脚手架

- 下面的程序包含了**所有**必需的部分。读注释,找出每一个部分。
- 它使用一个固定的列表,所以每次运行结果都一样(这里不需要现场输入)。
- 先运行它,然后修改数据和规则,把它变成**你自己的**。

---

```python
# 1) A LIST that stores many values
scores = [72, 95, 58, 88, 40, 67]

# 2) A PROCEDURE with a PARAMETER, that returns a result
def grade_for(score):
    # selection inside the procedure: choose a letter for one score
    if score >= 90:
        return "A"
    elif score >= 70:
        return "B"
    elif score >= 50:
        return "C"
    else:
        return "F"

# 3) An ALGORITHM: SELECTION (if) INSIDE ITERATION (for loop)
pass_count = 0
for s in scores:                 # iteration: visit every score
    letter = grade_for(s)        # call the procedure with a parameter
    if letter != "F":            # selection: is this a pass?
        pass_count = pass_count + 1
    print(f"score {s} -> grade {letter}")   # output

# More output: a summary
print(f"{pass_count} out of {len(scores)} passed")
```

## 把它变成你自己的

- 把**列表**换成你自己的数据(也试试不同的长度)。
- 修改**过程**:新的规则、新的参数名,或者返回别的东西。
- 修改**算法**:统计点新的东西,或者只保留通过 `if` 的那些项。
- 如果你愿意,可以加上**输入**,例如 `input("Enter a name: ")`。

## 书面回答题

- 围绕**你自己的程序**回答下面这些问题(真正的任务要求的就是这类写作):
- **目的:** 你的程序做什么?为什么它有用?
- **列表:** 你的列表存了什么?为什么用一个列表(而不是许多个单独的变量)会让程序**更容易管理**?

---

## 更多书面回答题

- **过程与参数:** 说出你的过程的名字。它的**参数**是什么?过程用这个参数**做了什么**?它返回什么?
- **算法:** 用你自己的话描述循环**内部**的算法。指出它在哪里用了**选择**(`if`),在哪里用了**迭代**(循环),并说明为什么两者都需要。

## 评分清单

- 当你的程序具备某一项后,就在它前面打勾:

---

```text
[ ] A list (or collection) is used in the program
[ ] A procedure is defined WITH at least one parameter
[ ] That procedure is CALLED somewhere in the program
[ ] The procedure does real work (not empty)
[ ] An algorithm uses selection (if) INSIDE iteration (a loop)
[ ] The program has input and/or output
[ ] I can explain every part in my own words
```
