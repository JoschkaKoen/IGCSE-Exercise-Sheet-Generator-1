## 为什么用列表?

- 程序常常一次需要**很多个值**,比如全班的分数。
- 一个值用一个变量会很乱。
- **列表**(list)用一个名字保存很多个值 —— 这是一个有用的**数据抽象**(data abstraction)。

---

## 什么是数据抽象?

- **抽象**(abstraction)隐藏细节,让你思考整体,而不是每个部分。
- 列表让你用**同样的方式**处理 1 个值或 1000 个值。
- 不管列表有多长,你写的代码都一样。

---

## 创建列表并读取它

- 把元素写在方括号里:`[10, 20, 30]`。
- 用**索引**(index,也就是位置)取出一个元素。Python 从 `0` 开始。
- `scores[0]` 是第一个元素;`scores[-1]` 是最后一个元素。

---

```python
scores = [10, 20, 30]
print(scores[0])
print(scores[-1])
print(len(scores))
```

---

## 用 append 让列表变长

- `scores.append(x)` 把 `x` 加到列表的**末尾**。
- 程序运行时,列表可以变长。
- `len(scores)` 告诉你现在有多少个元素。

---

```python
scores = [10, 20]
scores.append(30)
print(scores)
print(len(scores))
```

---

## 遍历(访问每一个元素)

- **遍历**(traverse)一个列表,就是一个接一个地走过每个元素。
- `for s in scores:` 每一轮给你一个元素 —— 这是 **for-each**(逐项)循环。
- 这是处理整个列表的简洁方法。

---

```python
scores = [10, 20, 30]
total = 0
for s in scores:
    total = total + s
print(total)
```

## 用 AP CSP 伪代码表示

- 列表也是考试里的 list —— 但 CB 的列表是**从 1 开始计数的**(1-indexed)。
- 所以在 CB 里 `aList[1]` 是**第一个**元素,而 Python 的第一个是 `scores[0]`。
- `FOR EACH x IN list` 对应 Python 的 `for x in list:`。`DISPLAY` = print,`←` = `=`。

```text
aList ← [10, 20, 30]
DISPLAY aList[1]        ## the FIRST item (1-indexed!)

total ← 0
FOR EACH x IN aList
{
    total ← total + x
}
DISPLAY total
```
