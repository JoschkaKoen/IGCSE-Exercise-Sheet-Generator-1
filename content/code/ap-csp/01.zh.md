## 每个程序都有目的

- **程序**是一组计算机要执行的指令。
- 每个程序都是为了某个**原因**而写的 —— 解决一个问题或帮助人们。
- 例如:发送一条消息、播放一首歌、找到最短的道路。

---

## 输入 → 处理 → 输出

- 大多数程序分三步工作。
- **输入**(input):数据进来(你输入的数字、一次点击、一个文件)。
- **处理**(process):程序对这些数据做处理。
- **输出**(output):结果出来(屏幕上的文字、一段声音、保存的文件)。

---

```text
INPUT            PROCESS                 OUTPUT
"5"     ->   add 5 and 3      ->        "8"
```

---

## 一个小例子

- 这个程序接收一个**输入**(名字),做一点小**处理**,然后显示**输出**。
- 你现在不用写它 —— 只要读一读,找出这三个步骤。

---

```python
name = input("Your name: ")     # input
greeting = "Hello, " + name     # process
print(greeting)                 # output
```

## 设计是一个循环

- 好的程序不是第一次就写得完美的。
- 我们使用一个**迭代**(iterative)循环:一遍又一遍地做,每次都好一点。
- **计划** → **编写** → **测试** → **改进**,然后重复。

---

```text
   plan ──> write ──> test ──> improve
    ^                              │
    └──────────────────────────────┘
```

---

## 出错是正常的

- **错误**(**bug**)是让程序做错事的一个小毛病。
- 在基础课程里你学过**调试**(debug):读报错信息、找到那一行、改正它。
- 找出并修复 bug 是循环中每一步都很正常的一部分。

---

## 计算是协作的

- 大多数真实的程序是由一个**团队**完成的,而不是一个人。
- 人们分享想法、分配工作、互相检查代码。
- 不同的人带来不同的视角,这让程序变得更好。
