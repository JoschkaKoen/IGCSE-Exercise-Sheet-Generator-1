## 上一课留下的问题

- 上一课里,函数拿到的是你那个数字的**副本**。改这个副本,**不会**改到原来的变量。
- C 会**复制**你传给函数的每一个参数。所以普通的函数没办法改你的变量。
- 要想改它,函数需要这个变量的**地址** —— 也就是它在内存里待的位置。

---

## 每个变量都有一个地址

- 把变量想成内存里的一个**盒子**,值就放在盒子里面。
- 每个盒子都有一个**地址**:一个数字,告诉你这个盒子*在哪里*。
- `&` 运算符就能取出这个地址。`&x` 表示“`x` 的地址”。

---

## 指针保存的是地址

- **指针**是一种变量,它保存的是一个**地址**,而不是普通的值。
- 用 `*` 来声明指针:`int *p = &x;` 读作“`p` 指向 `x`”。
- 要访问那个地址里的值,在前面加 `*`:`*p`。这叫做**解引用**。
- 改 `*p` 就会改到**原来的**变量。`*p = 5;` 会把 `5` 放进 `x` 本身。

---

```c
#include <stdio.h>

void set_to_five(int *p) {
    *p = 5;            // change the value at the address
}

int main(void) {
    int x = 0;
    set_to_five(&x);   // pass the ADDRESS of x
    printf("%d\n", x); // 5 — the original changed!
    return 0;
}
```

---

## 传出指针:一次给回多个答案

- 一个函数只能 `return` **一个**值。但它可以通过指针改动好几个变量。
- 你把每个答案的**地址**传进去,比如 `find(&min, &max)`。函数用 `*` 把结果写进去。
- 这些额外的指针参数叫做**传出指针**。调用之后,调用者就能读到这些答案。
- 一个不指向任何东西的指针是 `NULL`。绝对不要对 `NULL` 指针解引用。

---

- 每个任务都给你一个要补全的函数。用 `*` 来读或改指针所指向的值。
- 当一个值是通过指针传进来的(比如 `int *p`),把答案通过它写回去 —— 不要用 `return`。
