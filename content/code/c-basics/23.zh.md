## 在编译器之前运行的东西

- 在编译器看到你的代码之前,**预处理器**会先处理每一行以 `#` 开头的内容。
- 它把头文件粘贴进来、替换宏、去掉注释 —— 是一个简单的文本步骤。
- 你已经用过它了:`#include <stdio.h>` 就是一条预处理命令。

---

## #include 与 #define

- `#include <stdio.h>` 把一个头文件的内容粘贴进来,这样你就能用它的函数。
- `#define PI 3.14159` 把 `PI` 变成一个**宏**:预处理器会把每个 `PI` 替换成那段文本。
- 宏没有类型,也没有分号 —— 它纯粹是文本替换。

---

## const 与 #define

- `const double pi = 3.14159;` 是一个**真正的变量**,但不能被改变。它有类型,所以编译器会检查它。
- `#define PI 3.14159` 只是文本替换,在编译之前完成。
- 现代 C 通常更偏向用 `const` 表示数值,但 `#define` 在常量和标志位上仍然常见。

---

## static 与 enum

- 函数里的 `static` 变量会**在多次调用之间保留它的值** —— 它只创建一次,而不是每次函数运行时都重置。
- `enum` 给一组相关的 `int` 值起名字:`enum Suit { HEARTS, DIAMONDS, CLUBS, SPADES };`(这里 `HEARTS` 是 `0`)。
- 两者都让代码更清晰:`static` 用于需要保留的内存,`enum` 用于有名字的选项。

---

```c
#include <stdio.h>

#define GREETING "Hello"
enum Color { RED, GREEN, BLUE };

int main(void) {
    printf("%s\n", GREETING);   // Hello
    enum Color c = GREEN;
    printf("%d\n", c);          // 1
    return 0;
}
```

---

## 现在轮到你了

- 使用起始代码里给出的 `#define` 和 `enum`。
- 函数里的 `static` 局部变量会在多次调用之间记住它的值。**不要**自己写 `main` —— 检查器会提供。
