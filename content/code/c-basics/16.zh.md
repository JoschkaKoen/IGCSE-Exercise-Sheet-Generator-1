## 栈内存与堆内存

- 普通的局部变量住在**栈**上。它们在函数开始时出现,在函数返回时消失。
- 有时你需要的内存要**活得比函数久**,或者它的大小只有在运行时才知道。
- 这种内存来自**堆**。你向它要,之后再还回去。

---

## malloc 与 sizeof

- `malloc` 向堆要一块字节,并返回指向它的**指针**。
- 用 `sizeof` 取得一个元素的大小:`malloc(n * sizeof(int))` 为 `n` 个 int 腾出空间。
- 这块内存归你用,通过指针访问,就和数组一样:`a[0]`、`a[1]`……

---

## free 与内存泄漏

- 用完堆内存后,用 `free(指针)` 把它还回去。
- 如果你永远不 `free`,程序就会**泄漏**内存 —— 它占着不再使用的空间。
- 这些任务的规则:**你的函数负责分配并返回;检查器负责 free。**不要自己调用 `free`。

---

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    int *a = malloc(3 * sizeof(int));   // room for 3 ints
    a[0] = 5; a[1] = 6; a[2] = 7;
    printf("%d\n", a[0] + a[1] + a[2]); // 18
    free(a);                            // give it back
    return 0;
}
```

---

## 用 realloc 扩大数组

- `realloc(a, newSize)` 给堆上的一块内存改变大小,并保留原来的内容。
- 它可能会移动这块内存,所以一定要用它的**返回值**:`a = realloc(a, ...)`。
- 这就是一次加一个元素地扩大数组的方法,就像一个会变长的列表。

---

## 现在轮到你了

- 新数组用 `malloc(n * sizeof(int))`,并 `#include <stdlib.h>`。
- 你的函数返回指针;检查器会 free 它 —— 你**不要**调用 `free`。也**不要**写 `main`。
