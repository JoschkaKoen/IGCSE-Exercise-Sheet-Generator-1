## 三种错误

- **语法错误**(syntax error)违反了 Python 的规则,代码根本无法运行。
- **运行时错误**(runtime error)在程序**运行过程中**让它崩溃。
- **逻辑错误**(logic error)能正常运行,但给出**错误的**答案。

---

## 语法错误

- 这些是笔误:漏了 `:`、引号没闭合、缩进不对。
- Python 拒绝启动,并指给你看是哪一行。
- 修正拼写或标点,然后重新运行。

---

## 运行时错误

- 代码开始运行,然后碰到了做不到的事情。
- 例如:`10 / 0`,或 `int("hello")`,或读取超出列表末尾的位置。
- 程序会在那一行停下,并显示一条消息。

---

## 逻辑错误

- 程序一直运行到结束,但结果是错的。
- 例如:本想用 `+` 却用了 `-`,或循环停得太早。
- 这些最难找 —— 计算机完全照你写的去做了。

---

## 读懂回溯信息

- 程序崩溃时,Python 会打印一段**回溯信息**(traceback)。
- 先读**最后**一行:它说明了错误类型和出了什么问题。
- 行号会告诉你该去**哪里**查看。

```text
Traceback (most recent call last):
  File "<program>", line 2, in <module>
    print(10 / 0)
ZeroDivisionError: division by zero
```

---

## 用选定的数据测试

- 在测试之前,不要相信代码是对的。
- 用**正常**数据、**边界**数据(例如 `0`)和**极端**数据。
- 把每个答案和你手算的结果对照检查。

---

```python
def square(n):
    return n * n

print(square(3))    # normal -> 9
print(square(0))    # boundary -> 0
print(square(-2))   # negative -> 4
```

- 按**检查答案**,用多组数据测试你的修复。
