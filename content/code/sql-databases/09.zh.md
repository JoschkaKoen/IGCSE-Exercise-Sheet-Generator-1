# 不止一张表

真实的数据库把数据分散在**多张表**里。本课用一个小商店,它有两张表:

- `customer` —— 每位顾客一行(`id`、`name`、`city`)
- `orders` —— 每张订单一行(`id`、`customer_id`、`order_date`、`total`)

如果把顾客的名字存在每一张订单上,就会一遍遍地重复。把顾客单独放一张表,每位顾客只存**一次**。

---

# 主键

**主键**是**唯一标识**每一行的那一列。它从不为空,也从不重复。

- `customer.id` 是 `customer` 的主键。
- `orders.id` 是 `orders` 的主键。

给定一个主键值,你总能找到恰好一行。

---

# 外键

**外键**是一列,它保存**另一张**表的主键,把两张表连接起来。

`orders.customer_id` 保存的是来自 `customer.id` 的值 —— 它说明每张订单**属于哪位顾客**。

**参照完整性**指每个外键值都必须对应一个真实存在的主键:订单不能属于一个不存在的顾客。
