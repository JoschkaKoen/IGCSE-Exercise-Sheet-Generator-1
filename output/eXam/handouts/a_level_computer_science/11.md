## Programming basics

### From design to code

You should be able to turn a design вҖ” a **flowchart** жөҒзЁӢеӣҫ or **structured English** з»“жһ„еҢ–иӢұиҜӯ вҖ” into **pseudocode** дјӘд»Јз Ғ, and then into a real language:

1. find the **variables** еҸҳйҮҸ and their **data types** ж•°жҚ®зұ»еһӢ.
2. turn input/output boxes into `INPUT` / `OUTPUT`.
3. turn decision diamonds into `IF...ELSE...ENDIF` (or `CASE`).
4. turn loop arrows into `WHILE`, `REPEAT...UNTIL`, or `FOR`.
5. turn process boxes into assignments or calculations.
6. check by tracing a small input.

### Constants and variables

A **constant** еёёйҮҸ holds a value that **never changes**; a variable holds one that may change. Declare them with a type:

```
CONSTANT Pi вҶҗ 3.14159
DECLARE Radius : REAL
DECLARE Area : REAL

Radius вҶҗ 5
Area вҶҗ Pi * Radius * Radius
```

Use constants for fixed values that recur (`Pi`, `MaxScore`); they make code clearer and easy to change in one place.

### Assignment and expressions

Use `вҶҗ` for **assignment** иөӢеҖј:

```
Total вҶҗ Total + 1
Average вҶҗ Sum / Count
```

Expressions use **operators** иҝҗз®—з¬Ұ:

- arithmetic `+ - * /`, plus `DIV` (integer division) and `MOD` (remainder): `7 DIV 2 = 3`; `7 MOD 2 = 1`.
- comparisons `=`, `<>`, `<`, `>`, `<=`, `>=`.
- logic `AND`, `OR`, `NOT`.

**Precedence** дјҳе…Ҳзә§ (highest to lowest): `NOT` вҶ’ `* / DIV MOD` вҶ’ `+ -` вҶ’ comparisons вҶ’ `AND` вҶ’ `OR`. Use brackets when unsure.

### Input and output

```
OUTPUT "Enter your name:"
INPUT Name
OUTPUT "Hello, ", Name
```

### Built-in functions and library routines

Many tasks have ready-made **library routines** еә“дҫӢзЁӢ, so you need not write them:

- string: `LENGTH(s)`, `LEFT(s, n)`, `RIGHT(s, n)`, `MID(s, start, len)`, `UCASE(s)`, `LCASE(s)`.
- numeric: `INT(x)`, `ROUND(x)`, `ABS(x)`, `MOD(a, b)`, `RANDOM()`.
- conversion: `STR(x)` (number вҶ’ string), `VAL(s)` (string вҶ’ number).

Use the exact names from the question paper's reference list.

## Selection

**Selection** йҖүжӢ© chooses which steps run.

```
IF age >= 18 THEN
    OUTPUT "Adult"
ELSE
    OUTPUT "Minor"
ENDIF
```

For more than two cases you can use a **nested** еөҢеҘ— IF, but deep nesting is hard to read вҖ” a `CASE` is cleaner when testing one value against several options:

```
CASE OF Grade
    "A": OUTPUT "Excellent"
    "B": OUTPUT "Good"
    OTHERWISE: OUTPUT "Try again"
ENDCASE
```

Cambridge `CASE` allows single values, value lists (`1, 2, 3:`), and ranges (`1 TO 5:`).

## Iteration

**Iteration** иҝӯд»Ј repeats a block. Three loops differ in how many times the body runs.

### Count-controlled (FOR) loop

A **count-controlled loop** и®Ўж•°еҫӘзҺҜ вҖ” use it when you know how many times to repeat:

```
FOR i вҶҗ 1 TO 10
    OUTPUT i
NEXT i
```

A `STEP` can change the count (e.g. `FOR i вҶҗ 10 TO 1 STEP -1`). Best for a fixed number of repeats or processing each element of an **array** ж•°з»„.

### Pre-condition (WHILE) loop

A **pre-condition loop** еүҚжөӢеҫӘзҺҜ tests the condition **before** each pass, so it may run **zero** times:

```
WHILE total < 100 DO
    INPUT n
    total вҶҗ total + n
ENDWHILE
```

### Post-condition (REPEAT...UNTIL) loop

A **post-condition loop** еҗҺжөӢеҫӘзҺҜ tests the condition **after** each pass, so it always runs **at least once**:

```
REPEAT
    INPUT password
UNTIL password = correctPassword
```

### Choosing the right loop

- count known up front вҶ’ **FOR**.
- may need zero passes вҶ’ **WHILE**.
- always at least one pass вҶ’ **REPEAT...UNTIL**.

Justify your choice by whether the count is known and whether the body must run at least once. A typical question gives a scenario ("ask for a password until correct, but always ask at least once") and asks which loop fits.

## Procedures and functions

**Structured programming** з»“жһ„еҢ–зј–зЁӢ builds a program from small named **subroutines** еӯҗзЁӢеәҸ, each with one job.

### Procedure

A **procedure** иҝҮзЁӢ is a named block that **does an action**; it may take **parameters** еҸӮж•° but does **not return a value**.

```
PROCEDURE Greet(name : STRING)
    OUTPUT "Hello, ", name
ENDPROCEDURE

CALL Greet("Ada")
```

### Function

A **function** еҮҪж•° is like a procedure but it **returns a value** that becomes part of an expression.

```
FUNCTION Square(x : INTEGER) RETURNS INTEGER
    RETURN x * x
ENDFUNCTION

result вҶҗ Square(5) + 1     // result = 26
```

Use a **procedure** when the subroutine performs an action; use a **function** when it computes a value for the caller.

### Parameters

A **parameter** is a variable a subroutine declares to receive input; the values the caller supplies are **arguments** е®һеҸӮ. Two ways to pass them:

- **pass by value** дј еҖј вҖ” the routine gets a **copy**; changes inside it do not affect the caller. Use for inputs it only reads.
- **pass by reference** дј еј•з”Ё вҖ” the routine gets a **reference** to the caller's variable; changes **do** affect the caller. Use when it must update a parameter.

```
PROCEDURE Swap(BYREF a : INTEGER, BYREF b : INTEGER)
    DECLARE temp : INTEGER
    temp вҶҗ a
    a вҶҗ b
    b вҶҗ temp
ENDPROCEDURE
```

### Local vs global variables

A **local variable** еұҖйғЁеҸҳйҮҸ is declared inside a subroutine and exists only while it runs. A **global variable** е…ЁеұҖеҸҳйҮҸ is declared outside and is visible everywhere. Prefer locals and parameters вҖ” heavy use of globals makes code hard to follow and test. (The region where a name is visible is its **scope** дҪңз”Ёеҹҹ.)

### When to use a subroutine

Use one when the same logic appears in **more than one place**, when a block has a **clear named purpose**, when the program is **complex** (use **decomposition** еҲҶи§Ј), or when you want to **test a piece in isolation**. Don't make them so tiny that the call costs more than the content.

### Terminology

- **definition** вҖ” the `PROCEDURE ... ENDPROCEDURE` (or function) block.
- **call** вҖ” where it is invoked. **argument** вҖ” a value passed in. **parameter** вҖ” the variable that receives it.
- **return value** вҖ” what a function passes back.
- **signature** зӯҫеҗҚ вҖ” name + parameters + return type.

## Writing efficient pseudocode

- **move invariants out of loops** вҖ” if a value (an **invariant** дёҚеҸҳйҮҸ) does not change with the loop counter, compute it once before the loop.
- **exit a loop early** when the answer is found (stop a **linear search** зәҝжҖ§жҹҘжүҫ as soon as the target appears).
- **avoid redundant work** вҖ” store a result and reuse it instead of recomputing.
- **choose the right data structure** вҖ” an array beats many separate variables when the items belong together.
- **replace deep nested IFs with CASE** when testing one value against many.
- **comment the intent**, not the mechanics (`// validate the postcode`, not `// loop 6 times`).
- **use meaningful names** (`numberOfPupils`, not `n`) and **initialise variables** before use.
