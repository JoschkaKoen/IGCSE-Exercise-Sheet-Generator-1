## What AI is

**Artificial Intelligence (AI)** is the area of computer science that builds systems that perform tasks normally requiring human intelligence — recognising speech and images, translating language, playing games at expert level, driving vehicles, making decisions from data, generating text or images.

Modern AI is overwhelmingly built using **machine learning** — algorithms that learn patterns from data instead of being explicitly programmed step-by-step. Within machine learning, **deep learning** — using artificial neural networks with many layers — has become dominant since the 2010s.

## Graphs in AI

Many AI problems are naturally posed on **graphs** — a set of **nodes** (states, objects, locations) connected by **edges** (transitions, relationships, possible moves).

### Examples

- **Pathfinding**: roads form a graph; finding the shortest route from A to B is a graph-search problem. Algorithms like **Dijkstra** and **A*** explore the graph efficiently.
- **Game playing**: each board position is a node; each legal move is an edge. A computer can search the game tree (a tree is a special graph) to choose a strong move. **Minimax** with **alpha-beta pruning** is the classic algorithm; **Monte Carlo Tree Search** is used in modern game-playing AI (AlphaGo, etc.).
- **State-space search**: any planning problem can be modelled as moving between states by applying operators. The AI's job is to find a sequence of operators that reaches a goal state.
- **Knowledge representation**: a **semantic network** is a graph where nodes are concepts and edges are relationships ("Snowy IS-A dog", "dog IS-A animal"). Used in early AI to encode facts and reason about them.
- **Knowledge graphs**: large modern databases of facts about the world (search engines, virtual assistants) are graphs where nodes are entities and edges are relationships, plus algorithms that traverse them to answer queries.

Algorithms like **breadth-first search**, **depth-first search**, **best-first search** and **A*** are standard tools for navigating these graphs efficiently — finding paths, detecting cycles, identifying components.

## Artificial neural networks (ANNs)

An **artificial neural network** is a computational model inspired by the **biological neurons** of a brain. The basic unit is an **artificial neuron**:

- Receives several **input values** (one from each connection coming in).
- Multiplies each input by a corresponding **weight**.
- Adds the weighted inputs together with a **bias** term.
- Applies an **activation function** (a non-linear function — sigmoid, ReLU, tanh) to the sum.
- The result is the neuron's **output**, which becomes input for neurons further down the network.

Neurons are arranged in **layers**:

- **Input layer** — receives the raw data (one neuron per input feature).
- **Hidden layers** — one or more layers between input and output where the network learns useful internal representations.
- **Output layer** — produces the final result (a classification, a regression value, a probability distribution).

When the network has many hidden layers, it is a **deep neural network**, and training it is **deep learning**.

### How ANNs have helped machine learning

Neural networks let machine learning models **learn complex patterns** directly from raw data — pixels of an image, samples of audio, characters of text — without humans having to design hand-crafted features. This has produced breakthroughs in:

- **Image recognition** — recognising objects in photos approaches human accuracy.
- **Speech recognition** — accurate voice-to-text.
- **Machine translation** — fluent automated translation between languages.
- **Natural language processing** — sentiment analysis, summarisation, chatbots, code generation.
- **Game playing** — beating world champions at Go, poker, StarCraft.
- **Generative models** — producing realistic images, audio, text, video.

ANNs do well at problems with:

- large amounts of training data,
- noisy or high-dimensional input,
- patterns that are hard to express as explicit rules.

## Machine learning, deep learning, reinforcement learning

These three terms are related but distinct.

### Machine learning (ML)

The **umbrella term**: any algorithm that **learns from data** to improve its performance on a task. Three classical paradigms:

- **Supervised learning** — the training data has **labels** (e.g. images each tagged "cat" or "dog"). The algorithm learns a function from input to label. Used for classification, regression, prediction. Example: detecting spam emails from a labelled training set.
- **Unsupervised learning** — the data has **no labels**. The algorithm finds **structure** in the data: clusters, dimensions, anomalies. Example: grouping customers by shopping behaviour.
- **Reinforcement learning** (see below) — learns by trial and error.

When to use ML: when explicit rules would be impractical or impossible to write down. Spam filters, recommendation engines, fraud detection, medical diagnosis — all rely on patterns too complex for a programmer to encode by hand.

### Deep learning (DL)

A **subset of ML** that uses **neural networks with many layers** (deep neural networks). The deep architecture lets the model **learn hierarchical features** — lower layers detect simple patterns (edges, phonemes), higher layers combine them into more abstract concepts (shapes, words).

Deep learning is the technique behind most state-of-the-art results in vision, speech, language and games. It needs:

- **lots of data** (millions of examples often),
- **lots of compute** (typically GPUs or specialised hardware),
- careful design and tuning of the network architecture.

When to use DL: when the task is highly complex (vision, language) and you have lots of data and lots of compute. For small datasets simpler ML methods (logistic regression, decision trees, random forests) often work better.

### Reinforcement learning (RL)

The learner — called the **agent** — interacts with an **environment**. At each step, the agent picks an **action**, the environment changes state and returns a **reward** (positive or negative). The agent's goal is to learn a **policy** — a strategy that picks actions — that **maximises the cumulative reward over time**.

No labelled data is given up front. The agent learns by **trial and error**, gradually figuring out which actions in which states lead to good outcomes.

When to use RL:

- **Sequential decision** problems — game playing, robot control, autonomous driving, dynamic resource allocation, network routing.
- Problems where the **right answer for an action depends on the future**, not on an immediate label.

RL has produced famous results: superhuman game players (Atari games, Go, chess, StarCraft); robot locomotion; data centre cooling.

## Training an ANN: backpropagation

Training a neural network means **adjusting the weights** so the network's outputs match the desired outputs on the training data. The standard technique is **backpropagation of errors**, combined with an optimiser like **gradient descent**.

The process for each training example:

1. **Forward pass** — feed the input through the network; compute the output at each layer through to the output layer.
2. **Compute the error** — compare the output to the target ("desired") value. Use a **loss function** (mean squared error, cross-entropy, etc.) that gives a single number summarising how wrong the output is.
3. **Backward pass (backpropagation)** — propagate the error **backwards** through the network. For each weight, compute how much it contributed to the error (its **gradient**). This uses the chain rule of calculus to work out, layer by layer, how the loss changes with respect to each weight.
4. **Update the weights** — nudge each weight in the direction that **reduces the error**, by a small amount controlled by the **learning rate**. This is gradient descent.

Repeat for many examples and many passes over the data (**epochs**) until the error stops shrinking.

After training, the network's weights encode what it has learned. New inputs (not seen in training) are fed through the network in a single forward pass to get predictions.

Why "back" propagation: the error flows from the output layer **back** to the input layer, so the gradients can be computed efficiently in a single backward sweep — much faster than recomputing for each weight independently.

## Regression methods

Some ML tasks predict a **numeric value** (the price of a house, the temperature tomorrow, the time taken to finish a job). These are called **regression** problems (as opposed to classification, which predicts a category).

### Linear regression

The simplest method. Fit a **straight line** (or hyperplane in higher dimensions) through the data:

$$
y = m_{1} x_{1} + m_{2} x_{2} + \ldots + m_{n} x_{n} + c.
$$

Choose the coefficients $m_{i}$ and $c$ to **minimise the sum of squared errors** between the line's predictions and the actual values in the training data.

Used when:

- the relationship between inputs and output looks roughly linear,
- you want an interpretable model (each coefficient tells you the effect of one input).

### Non-linear regression

When data is not well described by a line, use models with more flexible shapes — **polynomial regression**, **decision-tree regression**, **neural-network regression** (the final layer of an ANN can output a numeric value).

The general idea is the same: define a parameterised model, define an error/loss function, and adjust the parameters to minimise the loss on training data. Backpropagation is one specific algorithm (used for neural-network regression); other regression problems can use closed-form solutions or different optimisers.

### Where regression fits in ML

Regression is the **continuous-output cousin of classification**. Both are **supervised** learning. Many ML libraries provide both regression and classification variants of the same algorithm (random-forest classifier, random-forest regressor). Choosing between them comes down to the question: is the answer a category (cat/dog/bird) or a number (price, temperature)?

## How AI is used in a real scenario

Several of the recent exam scenarios use AI for similar purposes. Recognising the pattern is half the answer:

**Customer identification at an automated shop** (camera-based):

1. The system has been trained on **many labelled face images** of customers.
2. When a person enters, the camera captures their face.
3. Image-recognition (deep learning) extracts a **face representation**.
4. The system compares the new face representation to those of registered customers (perhaps using a graph or vector-database structure for fast lookup).
5. The closest match identifies the customer; if no close match, the customer is denied entry.

**Reading text from images** (OCR):

1. Image recognition (deep learning) finds text regions in the camera image.
2. Optical character recognition (another deep-learning model) extracts the characters.
3. The resulting string can then be **translated** (yet another deep-learning model, the machine-translation system) into another language.
4. **Text-to-speech** (deep-learning model on audio) reads the translation aloud.

**Automated item-detection at checkout**:

1. Cameras observe the shelves.
2. Object-detection AI (deep learning, trained on labelled images of products) identifies which items have moved into a customer's basket.
3. The customer's account is charged for the recognised items.

Pattern: each task uses a **deep-learning model trained on labelled data** of the relevant kind (faces, text, products). Multiple such models can be combined into a pipeline. The intelligence is in the patterns learned during training — by the time the user interacts with the system, the model is fast, doing only forward-pass inference.
