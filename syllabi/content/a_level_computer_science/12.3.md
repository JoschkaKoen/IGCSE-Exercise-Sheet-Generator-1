# 12.3 Program Testing and Maintenance

1. Show understanding of ways of exposing and avoiding faults in programs
2. Locate and identify the different types of errors (syntax errors, logic errors, run-time errors)
3. Correct identified errors
4. Show understanding of the methods of testing available and select appropriate data for a given method (Including dry run, walkthrough, white-box, black-box, integration, alpha, beta, acceptance, stub)
5. Show understanding of the need for a test strategy and test plan and their likely contents
6. Choose appropriate test data for a test plan (Including normal, abnormal and extreme/boundary)
7. Show understanding of the need for continuing maintenance of a system and the differences between each type of maintenance (Including perfective, adaptive, corrective)
8. Analyse an existing program and make amendments to enhance functionality
